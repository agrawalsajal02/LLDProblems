# SOLID Principles

This package groups the five SOLID principles into separate folders so they are easier to revise before interviews.

Folders:
- `srp` for Single Responsibility Principle
- `ocp` for Open/Closed Principle
- `lsp` for Liskov Substitution Principle
- `isp` for Interface Segregation Principle
- `dip` for Dependency Inversion Principle

Quick memory hook:
- `S`: one reason to change
- `O`: extend behavior, avoid editing stable code
- `L`: subtype should behave like base type
- `I`: small focused interfaces
- `D`: depend on abstractions, not concrete details
