# Interview Patterns

This package groups the design patterns that show up most often in LLD interviews.

Patterns included:
- `creational/factory`
- `creational/builder`
- `creational/singleton`
- `structural/decorator`
- `structural/facade`
- `behavioral/strategy`
- `behavioral/observer`
- `behavioral/statemachine`
- `behavioral/chainofresponsibility`
- `behavioral/mediator`
- `behavioral/template`

Buckets:
- `creational`: object creation patterns
- `structural`: wrapping and composition patterns
- `behavioral`: runtime interaction and state-flow patterns

Rule of thumb:
- Use a pattern when it removes repeated branching or coordination pain.
- Do not force a pattern just to name-drop it in an interview.
