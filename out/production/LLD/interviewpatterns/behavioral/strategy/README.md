# Strategy

## Use when
- Same job can be done in multiple interchangeable ways.
- You want to replace `if/else` behavior branches with polymorphism.

## Do not use when
- Behavior will never vary.
- One or two branches are simpler than multiple classes.

## Memory hook
- "Context delegates work to a pluggable behavior."

## Interview triggers
- Pricing logic
- Dispatch logic
- Payment/discount algorithm
