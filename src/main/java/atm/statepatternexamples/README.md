# State Pattern Study Set

This folder keeps three other common interview-friendly examples of the State Design Pattern alongside ATM.

Examples:
- `vendingmachine`
- `elevator`
- `trafficlight`

How to remember State Pattern:
- same object
- different current state
- different allowed action
- state itself decides next transition

Hinglish memory line:
`object same rehta hai, par current state ke hisaab se behavior badal jata hai`
