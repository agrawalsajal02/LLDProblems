# Mediator

## Use when
- Many objects need to communicate, but direct object-to-object references are becoming messy.
- You want to centralize collaboration logic in one place.

## Do not use when
- Only two objects talk to each other directly.
- The mediator would become a giant god object with all business rules.

## Memory hook
- "Objects talk to mediator, mediator coordinates everyone."

## Interview triggers
- Chat room
- Air traffic control
- UI form coordination
- Marketplace matching
